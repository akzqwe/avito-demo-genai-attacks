# Avito GenAI Security Demo

Учебное веб-приложение к докладу **«Практическая безопасность GenAI-агентов»**.
На одной странице (админка модератора Avito) воспроизводится полная цепочка атаки
на AI-агента поддержки:

> **косвенная промпт-инъекция → опасный tool-call → XSS в админке → эксфильтрация сессии модератора**

Демо намеренно уязвимо и предназначено только для локального прогона на выступлениях
и воркшопах. **Не разворачивайте его на публичных хостах.**

## Что показывает

- **LLM01 — Prompt Injection.** Payload лежит в описании объявления, а не в чате
  пользователя. Атакующий ничего не «пишет боту».
- **LLM02 / LLM07 — System-prompt leakage.** Инъекция вынуждает агента раскрыть
  свой системный промпт и внутренние инструкции.
- **LLM06 — Sensitive Information Disclosure.** Через tool-call вытекает сессия
  модератора в лог атакующего.
- **A17 — Excessive Agency / небезопасные tool-calls.** У агента есть тул
  `add_internal_note`, чей вывод рендерится в админке без санитизации → XSS.

Четыре защитных слоя (санитизация ввода, allow-list тулов, изоляция trust-доменов,
CSP на странице админа) включаются чекбоксами — чтобы вживую показать, что именно
блокирует какую фазу атаки.

## Стек

- Python 3.10+, FastAPI, Uvicorn, Pydantic
- Mock-LLM в `backend/llm.py` — детерминированный, без внешних API-ключей;
  повторяет поведение реальной модели под indirect injection
- Ванильный JS + статические HTML-страницы (`/`, `/admin`, `/attacker`)
- Слайды собираются `python-pptx` скриптом

## Быстрый старт

```bash
./run.sh
```

Скрипт идемпотентен: создаёт `.venv`, ставит зависимости, поднимает Uvicorn
на `http://127.0.0.1:8000`. Три вкладки для демо:

- <http://localhost:8000/> — витрина объявлений + чат
- <http://localhost:8000/admin> — админка модератора (XSS-сток)
- <http://localhost:8000/attacker> — лог эксфильтрации

## Структура

```
backend/       FastAPI-приложение, mock-LLM, in-memory store, tools
static/        HTML/CSS фронта (user / admin / attacker)
docs/          спецификации и планы (в т.ч. OWASP LLM Top 10 mapping)
scripts/       генератор PPTX-слайдов
run.sh         bootstrap + запуск
```

## Оговорки

- Уязвимости в демо **преднамеренны**. Не копируйте паттерны в продакшн.
- Данные хранятся в памяти процесса; перезапуск/`POST /api/admin/reset` возвращает
  сид-состояние.
- Docker/PaaS-развёртывание не планируется.
